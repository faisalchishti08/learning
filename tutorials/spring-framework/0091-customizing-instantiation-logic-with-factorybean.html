Customizing instantiation logic with FactoryBean

← Back to checklist

1. What it is

FactoryBean<T> is a Spring interface that lets you hide complex object-creation code behind a bean. Instead of Spring calling a constructor, it calls your FactoryBean.getObject() method to produce the actual bean. The result of getObject() — not the FactoryBean itself — is what gets injected into other beans.

Think of it as a factory inside the container: you register the factory as a bean, but callers receive the product.

2. Why & when

Some objects can't be created simply by new MyService():

  • They require multi-step setup (connect, authenticate, then build).
  • They come from a fluent builder API with many mandatory options.
  • They are generated by a framework at runtime (e.g., JPA EntityManagerFactory, MyBatis mappers, JNDI lookups).

FactoryBean lets you wrap that complexity in one class and expose a clean bean. Spring Data, Spring Integration, Spring Security, and Hibernate all ship FactoryBean implementations for exactly this reason.

3. Core concept

Implement three methods:

MethodReturnsPurpose
getObject()TThe actual object to inject
getObjectType()Class<?>The type of T (for type-safe lookup)
isSingleton()booleantrue = cache the result; false = call getObject() every time

When you call ctx.getBean("myFactory") you get the product (T). To get the FactoryBean itself, prefix with &: ctx.getBean("&myFactory").

5. Runnable example

Level 1 — Basic

A FactoryBean that creates a MessageDigest (an object you can't simply new because it's obtained via a static factory method).

// FbBasic.java
import org.springframework.beans.factory.FactoryBean;
import org.springframework.context.annotation.*;
import java.security.MessageDigest;
import java.security.NoSuchAlgorithmException;

class DigestFactoryBean implements FactoryBean<MessageDigest> {
    private final String algorithm;

    DigestFactoryBean(String algorithm) { this.algorithm = algorithm; }

    @Override
    public MessageDigest getObject() throws NoSuchAlgorithmException {
        return MessageDigest.getInstance(algorithm);
    }

    @Override
    public Class<?> getObjectType() { return MessageDigest.class; }

    @Override
    public boolean isSingleton() { return true; }
}

@Configuration
class DigestCfg {
    @Bean
    public DigestFactoryBean sha256Digest() {
        return new DigestFactoryBean("SHA-256");
    }
}

public class FbBasic {
    public static void main(String[] args) {
        var ctx = new AnnotationConfigApplicationContext(DigestCfg.class);
        // getBean returns the MessageDigest, not the FactoryBean
        MessageDigest digest = ctx.getBean(MessageDigest.class);
        System.out.println("Algorithm: " + digest.getAlgorithm());

        // Use & to get the FactoryBean itself
        var fb = (DigestFactoryBean) ctx.getBean("&sha256Digest");
        System.out.println("FactoryBean class: " + fb.getClass().getSimpleName());
        ctx.close();
    }
}

How to run: java FbBasic.java

ctx.getBean(MessageDigest.class) transparently returns the SHA-256 MessageDigest produced by getObject(). The & prefix exposes the factory itself.

Level 2 — Intermediate

A ConnectionFactoryBean that simulates a multi-step connection setup (connect → authenticate → ready) and exposes only the ready Connection object.

// FbConnection.java
import org.springframework.beans.factory.FactoryBean;
import org.springframework.context.annotation.*;
import org.springframework.beans.factory.annotation.Autowired;

// Simulated connection object
class Connection {
    private final String dsn;
    Connection(String dsn) { this.dsn = dsn; }
    public void query(String sql) { System.out.println("[" + dsn + "] > " + sql); }
    public void close() { System.out.println("[" + dsn + "] closed"); }
}

class ConnectionFactoryBean implements FactoryBean<Connection> {
    private final String host;
    private final int port;
    private final String user;
    private final String pass;

    ConnectionFactoryBean(String host, int port, String user, String pass) {
        this.host = host; this.port = port; this.user = user; this.pass = pass;
    }

    @Override
    public Connection getObject() {
        System.out.println("1. Connecting to " + host + ":" + port);
        System.out.println("2. Authenticating as " + user);
        System.out.println("3. Connection ready");
        return new Connection(host + ":" + port);
    }

    @Override public Class<?> getObjectType() { return Connection.class; }
    @Override public boolean isSingleton() { return true; }
}

class UserRepository {
    @Autowired Connection conn;

    public void findAll() { conn.query("SELECT * FROM users"); }
}

@Configuration
class ConnCfg {
    @Bean
    public ConnectionFactoryBean connection() {
        return new ConnectionFactoryBean("db.prod.example.com", 5432, "app", "secret");
    }

    @Bean public UserRepository userRepo() { return new UserRepository(); }
}

public class FbConnection {
    public static void main(String[] args) {
        var ctx = new AnnotationConfigApplicationContext(ConnCfg.class);
        ctx.getBean(UserRepository.class).findAll();
        ctx.close();
    }
}

How to run: java FbConnection.java

UserRepository is autowired with a Connection — not with the ConnectionFactoryBean. All the multi-step setup (connect, authenticate) is hidden inside getObject(). Callers are shielded from the ceremony.

Level 3 — Advanced

A PooledConnectionFactoryBean that builds a pool of connections, supports isSingleton() = false to hand out a fresh lease each call, and implements DisposableBean to clean up.

// FbPool.java
import org.springframework.beans.factory.DisposableBean;
import org.springframework.beans.factory.FactoryBean;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.*;
import java.util.ArrayDeque;
import java.util.Deque;
import java.util.concurrent.atomic.AtomicInteger;

// Simulated pooled connection
class PooledConn {
    private static final AtomicInteger seq = new AtomicInteger();
    final int id = seq.incrementAndGet();
    public void execute(String sql) { System.out.println("Conn#" + id + " > " + sql); }
}

class ConnectionPool implements FactoryBean<PooledConn>, DisposableBean {
    private final int maxSize;
    private final Deque<PooledConn> pool = new ArrayDeque<>();

    ConnectionPool(int maxSize) {
        this.maxSize = maxSize;
        for (int i = 0; i < maxSize; i++) pool.push(new PooledConn());
        System.out.println("Pool initialised with " + maxSize + " connections");
    }

    @Override
    public PooledConn getObject() {
        var conn = pool.isEmpty() ? new PooledConn() : pool.pop();
        System.out.println("Leased Conn#" + conn.id + " (pool remaining: " + pool.size() + ")");
        return conn;
    }

    public void returnToPool(PooledConn c) {
        pool.push(c);
        System.out.println("Returned Conn#" + c.id + " (pool remaining: " + pool.size() + ")");
    }

    @Override public Class<?> getObjectType() { return PooledConn.class; }
    // false = each getBean() call produces a different lease
    @Override public boolean isSingleton() { return false; }

    @Override
    public void destroy() {
        System.out.println("Pool shutting down — " + pool.size() + " connections drained");
        pool.clear();
    }
}

class OrderService {
    @Autowired ConnectionPool pool;

    public void processOrder(int id) {
        var conn = pool.getObject();    // explicit lease
        conn.execute("INSERT INTO orders(id) VALUES (" + id + ")");
        pool.returnToPool(conn);
    }
}

@Configuration
class PoolCfg {
    @Bean public ConnectionPool connectionPool() { return new ConnectionPool(3); }
    @Bean public OrderService orderService() { return new OrderService(); }
}

public class FbPool {
    public static void main(String[] args) {
        var ctx = new AnnotationConfigApplicationContext(PoolCfg.class);
        var svc = ctx.getBean(OrderService.class);
        svc.processOrder(1);
        svc.processOrder(2);
        svc.processOrder(3);
        ctx.close(); // triggers pool.destroy()
    }
}

How to run: java FbPool.java

isSingleton() = false means getObject() is called every time a bean is requested, enabling a lease-per-call pattern. DisposableBean.destroy() fires on ctx.close(), draining the pool cleanly.

6. Walkthrough

Execution order for the Level 3 example:

  1. AnnotationConfigApplicationContext created — Spring scans PoolCfg, registers connectionPool and orderService as BeanDefinitions.
  2. Spring recognises ConnectionPool implements FactoryBean — it marks the bean definition as a factory-backed type.
  3. ConnectionPool (the factory) instantiated — constructor new ConnectionPool(3) runs, creating 3 PooledConn objects and pushing them onto the deque. Pool size = 3.
  4. OrderService instantiated — the @Autowired ConnectionPool pool field is injected with the ConnectionPool factory itself (not with a PooledConn), because the field type is ConnectionPool.
  5. processOrder(1) called — pool.getObject() pops Conn#3 from the deque (stack-order, last in = first out). Remaining = 2. The SQL is executed. returnToPool pushes Conn#3 back. Remaining = 3.
  6. Same pattern for orders 2 and 3 — each borrows and returns.
  7. ctx.close() — Spring calls destroy() on ConnectionPool (because it implements DisposableBean). The pool logs its shutdown.

Expected output:

Returned Conn#3 (pool remaining: 3)

Returned Conn#3 (pool remaining: 3)
Leased Conn#3 (pool remaining: 2)
Conn#3 > INSERT INTO orders(id) VALUES (3)

7. Gotchas & takeaways

ctx.getBean("myFactory") returns the product, not the FactoryBean. To get the factory itself use the & prefix: ctx.getBean("&myFactory"). Forgetting this is the most common FactoryBean mistake.
If isSingleton() returns true, getObject() is called once and cached. If it returns false, it's called on every getBean() call — make sure getObject() is thread-safe if the context is used concurrently.
  • FactoryBean is the classic Spring way to encapsulate complex instantiation; in modern apps @Bean methods on @Configuration classes can replace most use cases — prefer them for new code.
  • Real framework FactoryBean examples: SqlSessionFactoryBean (MyBatis), LocalContainerEntityManagerFactoryBean (JPA), AbstractRemoteSlsbInvokerInterceptor.
  • getObjectType() drives type-based getBean() and @Autowired resolution; returning null prevents type matching.
  • Implement InitializingBean or DisposableBean on the FactoryBean for lifecycle hooks on the factory itself.
