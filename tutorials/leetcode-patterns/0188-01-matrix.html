01 Matrix

← Back to checklist

1. What it is

Given a binary grid mat of 0s and 1s, return a grid of the same size where each cell holds the distance to the NEAREST 0. Example: mat = [[0,0,0],[0,1,0],[1,1,1]] → [[0,0,0],[0,1,0],[1,2,1]].

2. Why & when

This is Walls and Gates with 0 cells playing the role of gates and every cell (not just walls/rooms) needing a distance. It is the cleanest form of multi-source BFS: every 0 seeds the search at once, and BFS naturally computes each cell's distance to its nearest 0.

3. Core concept

Key idea: enqueue every 0 cell at distance 0. BFS expands outward in rings; the first time a 1 cell is reached, that ring number is its distance to the nearest 0, which gets written into the result grid.

Steps:

  1. Build a result grid initialized to 0 for cells that are already 0 in the input.
  2. Enqueue every 0 cell into the BFS queue, and mark it visited.
  3. Run BFS layer by layer; for each cell dequeued, examine its 4 neighbors.
  4. If a neighbor is unvisited, mark it visited, set its result value to current distance + 1, and enqueue it.
  5. Continue until the queue empties; every 1 cell will have been reached, since the grid is finite and connected through 4-directional steps.

Why it is correct: multi-source BFS explores cells in increasing order of distance from the nearest 0, because all 0s start simultaneously at distance 0. The first (and only) time a 1 cell is dequeued or discovered, its assigned distance is guaranteed to be the minimum over all 0s, since a shorter path — if one existed — would have reached it in an earlier ring.

4. Diagram

Every 0 seeds the BFS simultaneously; each 1 cell's distance is set the moment the nearest ring reaches it.

5. Runnable example

// ZeroOneMatrix.java
import java.util.*;

public class ZeroOneMatrix {

    // Level 1 -- Brute force: for every 1 cell, scan the whole grid
    // for the nearest 0 by computing distances to all 0 cells and
    // taking the minimum. Correct, but O((rows*cols)^2) -- comparing
    // every cell against every 0 cell directly.

    // KEY INSIGHT: multi-source BFS computes "distance to nearest 0"
    // for every cell in ONE linear pass, because the ring that reaches
    // a cell FIRST is guaranteed to come from the closest 0.

    // Level 2 -- Optimal: multi-source BFS from all 0 cells.
    static int[][] updateMatrix(int[][] mat) {
        int rows = mat.length, cols = mat[0].length;
        int[][] result = new int[rows][cols];
        boolean[][] visited = new boolean[rows][cols];
        Queue<int[]> queue = new LinkedList<>();

        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                if (mat[r][c] == 0) {
                    queue.add(new int[]{r, c});
                    visited[r][c] = true;
                }
            }
        }

        int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};
        while (!queue.isEmpty()) {
            int[] cell = queue.poll();
            for (int[] d : dirs) {
                int nr = cell[0] + d[0], nc = cell[1] + d[1];
                if (nr < 0 || nr >= rows || nc < 0 || nc >= cols || visited[nr][nc]) continue;
                visited[nr][nc] = true;
                result[nr][nc] = result[cell[0]][cell[1]] + 1;
                queue.add(new int[]{nr, nc});
            }
        }
        return result;
    }

    // Level 3 -- Hardened: a grid with all 0s returns an all-zero
    // result directly (every cell seeds the queue, nothing left to
    // expand into); the problem guarantees at least one 0 exists, so
    // no unreachable-cell case needs handling.

    public static void main(String[] args) {
        int[][] result = updateMatrix(new int[][]{{0,0,0},{0,1,0},{1,1,1}});
        for (int[] row : result) System.out.println(Arrays.toString(row));
        // [0, 0, 0]
        // [0, 1, 0]
        // [1, 2, 1]
    }
}

How to run: java ZeroOneMatrix.java

6. Walkthrough

Trace mat = [[0,0,0],[0,1,0],[1,1,1]]:

LayerCells reachedValue written
seedall 0 cells: (0,0),(0,1),(0,2),(1,0),(1,2)0
1(1,1) via (1,0)/(1,2)/(0,1); (2,0) via (1,0); (2,2) via (1,2)1
2(2,1) via (1,1), (2,0), or (2,2)2

Final grid matches [[0,0,0],[0,1,0],[1,2,1]]. Time complexity is O(rows × cols), since each cell enters the queue exactly once; space is O(rows × cols) for the queue and visited array.

7. Gotchas & takeaways

Gotcha: initializing the result grid with a large "infinity" sentinel and forgetting to special-case 0 cells (setting their result to 0 directly, via seeding them at distance 0) can leave the wrong value at cells that are themselves already 0.
  • The pattern is identical to Walls and Gates — 0 cells here play exactly the role gates played there, and 1 cells play the role of empty rooms.
  • Marking visited at ENQUEUE time (not dequeue time) is what prevents multiple 0 sources from double-writing the same target cell with inconsistent distances.
  • Related problems: Walls and Gates (identical structure, different labels), As Far from Land as Possible (same multi-source BFS, tracks a maximum instead of writing per-cell values).
