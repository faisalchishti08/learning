LeetCode Patterns › 14. Bitwise XOR / Bit Manipulation

Find the Difference

← Back to checklist

1. What it is

You are given two strings s and t, where t is generated by shuffling s and then adding one extra random lowercase letter at a random position. Find that extra letter. Example: s = "abcd", t = "abcde" → "e".

2. Why & when

This is Single Number's XOR-cancellation trick, applied to CHARACTERS instead of integers: every letter in s has a matching letter in t, except for the one extra letter, which is unpaired. Use this shape whenever a problem describes one collection as "the same as another, plus one extra item," regardless of whether the items are numbers or characters.

3. Core concept

Key idea: XOR every character of s AND every character of t together into one accumulator (treating each char as its underlying numeric code). Every letter that appears in both s and t cancels out in pairs (c ^ c = 0), leaving only the extra letter from t that had no match in s.

Steps:

  1. Initialize result = 0 (as an int, to hold a character code).
  2. For each character c in s: result ^= c.
  3. For each character c in t: result ^= c.
  4. After both loops, result holds the character code of the extra letter; cast it back to char and return.

Why it is correct: because t is s shuffled plus one extra letter, every character in s has EXACTLY one matching occurrence in t (accounting for duplicate letters correctly, since XOR-cancellation naturally handles repeated values the same way Single Number does). XOR-ing all of s's characters and all of t's characters together cancels every matched pair to 0, leaving only the character that appeared an odd number of times overall — the one extra letter in t.

5. Runnable example

// FindTheDifference.java
public class FindTheDifference {

    // Level 1 -- Brute force: count occurrences of each letter in s
    // into an int[26] array, then subtract counts of t's letters; the
    // letter whose count goes negative (or the one left over) is the
    // extra letter. Correct, but uses O(26) extra space and two
    // separate passes plus explicit count bookkeeping.

    // KEY INSIGHT: this is Single Number's XOR-cancellation, applied
    // to characters instead of integers -- every matched letter
    // between s and t cancels, leaving only the extra one.

    // Level 2 -- Optimal: XOR every character of both strings.
    static char findTheDifference(String s, String t) {
        int result = 0;
        for (char c : s.toCharArray()) result ^= c;
        for (char c : t.toCharArray()) result ^= c;
        return (char) result;
    }

    // Level 3 -- Hardened: works unchanged when the extra letter is a
    // DUPLICATE of a letter already in s (e.g. s="aa", t="aaa"), since
    // XOR-cancellation handles repeated values correctly regardless of
    // how many times they appear, as long as parity (odd/even count)
    // is preserved.

    public static void main(String[] args) {
        System.out.println(findTheDifference("abcd", "abcde"));
        // e
        System.out.println(findTheDifference("", "y"));
        // y
    }
}

How to run: java FindTheDifference.java

Final result casts to 'e', the extra letter. Time complexity is O(|s| + |t|), one pass over each string. Space is O(1).

7. Gotchas & takeaways

Gotcha: this technique relies on t containing s's letters PLUS exactly one extra letter (as the problem guarantees) — if the problem instead allowed t to be missing letters too (not just adding one), plain XOR-cancellation would no longer isolate a single, unambiguous answer.
  • This problem confirms that XOR-cancellation works on ANY primitive type reducible to an integer code, not just int values — char works identically, since Java automatically widens a char to its numeric code in a bitwise expression.
  • The count-array alternative (Level 1) is a reasonable and common solution too, and is worth mentioning as the more "obvious" approach before presenting the tighter XOR trick.
  • Related problems: Single Number (the identical technique applied to integers), Missing Number (a related XOR trick that cancels indices against values instead of two full collections).
