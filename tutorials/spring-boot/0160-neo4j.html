Neo4j

1. What it is

Neo4j is a graph database that stores data as nodes and relationships (edges), each with properties. Spring Boot auto-configures Neo4j via spring-boot-starter-data-neo4j, providing a Driver (Java Bolt driver), a Neo4jTemplate, and Spring Data Neo4j repositories. Connection is configured with spring.neo4j.uri and spring.neo4j.authentication.username/password. Data is queried with Cypher, Neo4j's declarative graph query language.

2. Why & when

Use Neo4j when:

  • Relationships are first-class — social networks (friends-of-friends), recommendation engines (users who bought X also bought Y), dependency graphs, knowledge graphs.
  • Deep traversal — finding all paths between nodes of arbitrary depth is O(log n) in a graph DB vs. exponential join costs in SQL.
  • Fraud detection — connected-entity analysis (shared addresses, shared devices) across millions of entities.

Avoid Neo4j for heavy aggregate queries (SUM, GROUP BY across many rows), bulk analytical workloads, or simple CRUD with no relationship traversal.

3. Core concept

Node: (:Person {name: "Alice"})      ← label + properties
Relationship: -[:KNOWS {since: 2020}]→  ← type + properties
Graph: (:Person)-[:KNOWS]->(:Person)

Spring Data Neo4j maps Java classes to Neo4j nodes:

@Node("Person")
class Person {
    @Id @GeneratedValue Long id;
    String name;
    @Relationship(type = "KNOWS") List<Person> friends;
}

Cypher query: MATCH (p:Person)-[:KNOWS]->(f:Person) WHERE p.name=$name RETURN f

4. Diagram

Graph traversal: Alice -[:KNOWS]-> Bob, Carol; Bob -[:BOUGHT]-> Widget. Find what Alice's friends bought in one Cypher query.

5. Runnable example

// Neo4jApp.java — Spring Boot project with spring-boot-starter-data-neo4j
// application.properties:
//   spring.neo4j.uri=bolt://localhost:7687
//   spring.neo4j.authentication.username=neo4j
//   spring.neo4j.authentication.password=password
// Start Neo4j: docker run -p 7474:7474 -p 7687:7687 -e NEO4J_AUTH=neo4j/password neo4j:5

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.data.neo4j.core.schema.*;
import org.springframework.data.neo4j.repository.Neo4jRepository;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@SpringBootApplication
public class Neo4jApp {
    public static void main(String[] args) {
        SpringApplication.run(Neo4jApp.class, args);
    }
}

@Node("Person")
class Person {
    @Id @GeneratedValue Long id;
    String name;
    @Relationship(type = "KNOWS", direction = Relationship.Direction.OUTGOING)
    List<Person> friends = new java.util.ArrayList<>();
    Person() {}
    Person(String name) { this.name = name; }
    public Long getId() { return id; }
    public String getName() { return name; }
    public List<Person> getFriends() { return friends; }
}

interface PersonRepo extends Neo4jRepository<Person, Long> {
    List<Person> findByName(String name);
}

@RestController
@RequestMapping("/people")
class PersonController {

    private final PersonRepo repo;

    PersonController(PersonRepo repo) { this.repo = repo; }

    @PostMapping
    public Person create(@RequestBody Person p) { return repo.save(p); }

    @GetMapping
    public List<Person> all() { return repo.findAll(); }

    @GetMapping("/{name}")
    public List<Person> byName(@PathVariable String name) {
        return repo.findByName(name);
    }
}

How to run:

  1. Start Neo4j: docker run -p 7474:7474 -p 7687:7687 -e NEO4J_AUTH=neo4j/password neo4j:5
  2. Add spring-boot-starter-data-neo4j to pom.xml, start the app.
  3. curl -X POST http://localhost:8080/people -H 'Content-Type: application/json' -d '{"name":"Alice","friends":[]}'
  4. curl http://localhost:8080/people → all Person nodes.

6. Walkthrough

  • spring-boot-starter-data-neo4j pulls in the Neo4j Java driver and Spring Data Neo4j. Neo4jAutoConfiguration creates a Driver (Bolt connection pool) from spring.neo4j.uri. Neo4jDataAutoConfiguration registers Neo4jTemplate and enables repository scanning.
  • @Node("Person") declares this class as a Neo4j node with the label Person. @Id @GeneratedValue lets Neo4j assign an internal element ID (Neo4j 5+) or a generated long.
  • @Relationship(type = "KNOWS", direction = OUTGOING) maps the friends list to Neo4j relationships of type KNOWS. Spring Data Neo4j hydrates the list by traversing those relationships when loading a Person.
  • Neo4jRepository<Person, Long> extends CrudRepository with Neo4j-specific methods. findByName generates a Cypher MATCH (p:Person) WHERE p.name = $name RETURN p.
  • repo.save(p) issues MERGE or CREATE Cypher statements to persist node and relationship changes.

7. Gotchas & takeaways

@GeneratedValue on Neo4j 5+ uses Neo4j's internal element ID (string), not a database sequence. Storing the ID as Long works with Neo4j's legacy integer IDs; migrate to String + @GeneratedValue(generatorClass = UUIDStringGenerator.class) for new projects.
Loading a Person with @Relationship List<Person> friends eagerly fetches all connected nodes — avoid infinite depth by using @Relationship on DTOs or limiting depth with Neo4jTemplate.findById(..., 1).
  • Cypher queries via @Query("MATCH (p:Person)-[:KNOWS*2]->(f) WHERE p.name=$name RETURN f") enable variable-depth traversal unavailable via derived queries.
  • spring.neo4j.connection-timeout and spring.neo4j.max-transaction-retry-time tune the Bolt driver behaviour.
  • For reactive Neo4j (WebFlux), use ReactiveNeo4jRepository — the driver supports reactive streams via Project Reactor.
  • Neo4j Browser (port 7474) provides a visual graph UI useful for development and debugging.
