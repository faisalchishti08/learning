# Design: CS & Interview Prep Subjects (System Design, Data Structures, LeetCode Patterns)

**Date:** 2026-07-14
**Status:** Approved (coverage skeleton), ready for implementation plan

## Goal

Add three new subjects to the existing Spring/Java learning checklist site, following the
exact same architecture as existing subjects (Microservices, GenAI, WebDev, etc.):

1. **System Design** — concepts + use-case designs, Java/Spring oriented
2. **Data Structures** — all DS concepts, Java-based
3. **LeetCode Patterns** — one section per pattern, named LeetCode problems as items

Deliverable now: **all microtopics enumerated and plugged into the UI** (index cards +
per-subject checklist pages + tutorial/manifest scaffolding). **No tutorial content is
written now** — content markdown is generated later; the pipeline already picks it up.

## Non-Goals

- Writing tutorial `.md` content (deferred; generated later per existing daily flow).
- New rendering/templating code. Reuse `shell.render`, `tutorial_shell`, `manifest`.
- Touching existing subjects or `java.html`.

## Architecture (existing, reused as-is)

Each subject is a Python dict:

```python
{
  "file": "system-design.html", "title": "System Design", "logo": "SD",
  "cat": "CS & Interview Prep",
  "subtitle": "...",
  "sections": [
    {"name": "1. Fundamentals & Concepts", "tag": "concepts", "groups": [
      {"g": "Group name", "items": ["microtopic", "microtopic", ...]},
      ...
    ]},
    ...
  ],
}
```

Pipeline (`_build/generate.py`):
1. Imports every `data_*` module, extends `PROJECTS` from each module's `PROJECTS` list.
2. `write_project` → `shell.render(...)` → writes `<file>` checklist HTML into repo root.
3. `build_index` → renders index cards grouped by `cat`, in `PROJECTS` order.
4. `build_tutorials` → for each project, if `content/<stem>/*.md` exists, builds
   `tutorials/<stem>/*.html`. Empty content dir = no tutorials built (our case now).
5. `manifest.build` → phase manifest driving incremental tutorial generation. Auto-includes
   new subjects because it iterates `PROJECTS`.

Topic identity: `topics.enumerate_topics` assigns a stable global index `gi` (1-based,
in section→group→item order) and a slug. Tutorial pages are named `%04d-<slug>.html` by `gi`.

## Coverage Skeletons

Groups are representative; implementation enumerates every `item` (microtopic). `[+Spring]`

### Subject 1 — System Design (`system-design.html`, logo `SD`)

Concept sections (each `{name, tag, groups}`):

1. Fundamentals & Estimation — what/why system design, requirements gathering (functional /
   non-functional), back-of-envelope estimation, latency numbers every engineer should know,
   QPS/storage/bandwidth math, SLA/SLO/SLI, tradeoff thinking.
2. Networking & Communication — DNS, IP, TCP vs UDP, HTTP/1.1/2/3, TLS, REST, RPC/gRPC,
   WebSocket, Server-Sent Events, long/short polling, webhooks, connection pooling. `[+Spring]`
3. Load Balancing & Reverse Proxy — L4 vs L7, algorithms (round-robin, least-conn, hashing),
   sticky sessions, health checks, reverse proxy, API gateway. `[+Spring Cloud Gateway]`
4. Caching & CDN — cache-aside/read-through/write-through/write-back, eviction (LRU/LFU/TTL),
   cache invalidation, hot keys, thundering herd, CDN, edge caching, Redis/Memcached. `[+Spring Cache]`
5. SQL Databases — relational model, normalization, indexing (B-tree/hash/composite/covering),
   query optimization, transactions, ACID, isolation levels, locking, connection pooling. `[+Spring Data JPA]`
6. NoSQL & Polyglot Persistence — key-value, document, wide-column, graph, time-series,
   BASE, denormalization, when to choose which.
7. Data Partitioning & Sharding — horizontal/vertical partitioning, sharding strategies
   (range/hash/directory), consistent hashing, hot spots, rebalancing, resharding.
8. Replication & Consistency — leader-follower, multi-leader, leaderless, sync/async,
   quorum, CAP theorem, PACELC, consistency models (strong/eventual/causal/read-your-writes),
   conflict resolution (LWW/vector clocks/CRDT).
9. Messaging & Async Processing — message queues, pub/sub, brokers, delivery semantics
   (at-most/at-least/exactly-once), ordering, backpressure, dead-letter, event streaming. `[+Kafka/RabbitMQ]`
10. Scalability & Performance — vertical vs horizontal, statelessness, throughput vs latency,
    percentiles (p50/p95/p99), profiling, N+1, batching, connection reuse.
11. Reliability, Availability & Resiliency — redundancy, failover, health checks, circuit
    breaker, retry, timeout, bulkhead, rate limiting, graceful degradation, chaos. `[+Resilience4j]`
12. Rate Limiting & Throttling — token bucket, leaky bucket, fixed/sliding window,
    distributed rate limiting, per-user/global quotas. `[+Bucket4j / Gateway RateLimiter]`
13. Storage Systems — object/blob storage (S3-like), block storage, distributed file systems,
    data lake/warehouse, cold vs hot storage.
14. Search & Indexing — inverted index, tokenization, ranking, autocomplete/typeahead,
    fuzzy search. `[+Elasticsearch / Spring Data Elasticsearch]`
15. Concurrency & Coordination — locks, optimistic vs pessimistic, distributed locks,
    leader election, ZooKeeper/etcd, idempotency keys, distributed transactions, 2PC/3PC.
16. Security & Auth at Scale — authN vs authZ, sessions vs tokens, OAuth2/OIDC, JWT, TLS/mTLS,
    secrets management, encryption at rest/in transit, DDoS protection. `[+Spring Security]`
17. Observability — structured logging, metrics, distributed tracing, correlation IDs,
    dashboards, alerting, on-call/runbooks. `[+Micrometer / Micrometer Tracing]`
18. Architecture Patterns — monolith, modular monolith, microservices, event-driven, CQRS,
    event sourcing, saga, outbox, hexagonal/ports-and-adapters, BFF, strangler fig.
19. API Design — REST maturity, resource modeling, versioning, pagination, filtering,
    idempotency, error contracts, HATEOAS, GraphQL, gRPC, rate-limit headers.
20. Deployment & Infrastructure — containers, Kubernetes, blue-green, canary, feature flags,
    multi-region/geo-distribution, disaster recovery, IaC, cloud primitives.

Use-case section (`21. Use-Case Designs`, following the 20 concept sections above),

design steps: Functional requirements / Non-functional requirements / Capacity estimation /

URL Shortener · Rate Limiter · Pastebin · Web Crawler · Notification/Push system · News Feed ·

File Storage & Sync (Dropbox/Drive) · Distributed Unique ID Generator · Leaderboard/Gaming ·
Ad-Click Aggregator / Analytics · Airbnb / Hotel Booking · Food Delivery (DoorDash) ·

### Subject 2 — Data Structures (`data-structures.html`, logo `DS`)

Groups per structure = Concept / Operations & Complexity / **Java implementation** /
Variants / Applications (adapt per structure). Sections:

1. Foundations — ADT vs data structure, Big-O/Big-Theta/Big-Omega, amortized analysis,
   space complexity, Java memory model basics, references vs primitives, autoboxing, arrays
   in the JVM.
2. Arrays & Dynamic Arrays — static arrays, multi-dimensional arrays, `ArrayList` internals,
   resizing/amortization, capacity, `Arrays` utilities, copy/fill/sort.
3. Strings — `String` immutability, `char[]`, `StringBuilder`/`StringBuffer`, string pool /
   interning, common operations, complexity of concatenation.
4. Linked Lists — singly, doubly, circular, sentinel/dummy nodes, `LinkedList`, operations,
   reversal, cycle detection.
5. Stacks — LIFO, array vs linked implementation, `ArrayDeque` as stack (vs legacy `Stack`),
   applications (parsing, undo, call stack).
6. Queues & Deques — FIFO, circular buffer, `ArrayDeque`, `Queue`/`Deque` interfaces,
   priority queue, monotonic queue, blocking queues overview.
7. Hashing — hash functions, collision resolution (chaining/open addressing), `HashMap`
   internals (buckets, load factor, resize, treeify), `HashSet`, `LinkedHashMap`, `equals`/
   `hashCode` contract.
8. Trees — terminology, binary tree, BST, traversals (in/pre/post/level), balanced trees,
   AVL, red-black, `TreeMap`/`TreeSet` (red-black backed), N-ary/general trees.
9. Heaps / Priority Queues — binary heap, heapify, `PriorityQueue`, min/max heap, d-ary heap,
   heap sort.
10. Tries — prefix tree, insert/search/prefix, compressed/radix trie, ternary search tree,
    applications (autocomplete, spell-check).
11. Graphs — representations (adjacency list/matrix/edge list), directed/undirected/weighted,
    modeling in Java, traversals (BFS/DFS), degree, cycles, connectivity.
12. Advanced Trees & Range Structures — segment tree, Fenwick / Binary Indexed Tree,
    interval tree, B-tree / B+-tree, suffix tree/array, k-d tree.
13. Disjoint Set / Union-Find — operations, union by rank/size, path compression, applications.
14. Probabilistic & Specialized — skip list, Bloom filter, LRU/LFU structures,
    circular buffer, sparse structures.
15. Java Collections Framework — interface hierarchy (`Collection`/`List`/`Set`/`Queue`/`Map`),
    implementations map, `Iterator`/`ListIterator`, `Comparable` vs `Comparator`,
    `Collections` utilities, fail-fast vs fail-safe, concurrent collections overview,
    immutable/unmodifiable collections.
16. Complexity Cheat-Sheet & Selection — time/space table across structures, choosing the
    right structure by access pattern, common pitfalls.

Islands · Two Heaps · Subsets (combinations & permutations) · Modified Binary Search ·
Bit Manipulation / XOR · Top-K Elements (heap) · K-way Merge · Backtracking ·
DP: 0/1 Knapsack · DP: Unbounded Knapsack · DP: Fibonacci / Linear · DP: Longest Common
Subsequence · DP: Palindromic Subsequence · DP: Longest Increasing Subsequence · DP: Grid /
Matrix · DP: Interval / Partition · DP: State Machine (stock problems) · Greedy ·
Monotonic Stack · Prefix Sum · Trie problems · Union-Find problems · Topological Sort ·
Shortest Path (Dijkstra / Bellman-Ford / Floyd-Warshall) · Segment Tree / BIT problems ·
Design (LRU / LFU / iterators / data-structure design) · Math & Geometry.

## Risks / Notes

- LeetCode problem names are public problem titles used as checklist labels — not reproducing
  problem statements or content.
- Slug collisions: `topics.slugify` truncates to 60 chars; two long distinct titles could
  collide within a card and would overwrite tutorial filenames later. Implementation should
  scan each subject for duplicate slugs and disambiguate labels if any collide.
- Large item counts increase page size but the existing pages already hold 500–1100 topics,
  so no new rendering concern.
