## 1. What it is

The `/actuator/info` endpoint returns a JSON document describing the application — version, build timestamp, git commit, environment details, or any custom metadata you choose to expose. Spring Boot auto-populates it from `build-info.properties`, `git.properties`, and `info.*` properties in `application.properties`. You extend it by implementing `InfoContributor`, which adds structured sections to the document.

## 2. Why & when

`/actuator/info` answers the ops question "which version of this service is running?" without needing SSH access or reading container labels:
- Support tickets: "Is production running version 2.3.1 or 2.4.0?"
- Post-deploy check: "Did the correct build land?"
- Automated version verification in CI/CD pipelines.

Write a custom `InfoContributor` when you want to expose application-specific metadata: feature flags, license info, tenant configuration, upstream dependency versions.

## 3. Core concept

Built-in info contributors (enabled by `management.info.*`):
- **`env`**: exposes `info.*` properties from `application.properties`. `management.info.env.enabled=true`.
- **`build`**: reads `build-info.properties` (generated by Maven/Gradle plugin). Enabled automatically when the file is present.
- **`git`**: reads `git.properties` (generated by git-commit-id plugin). `management.info.git.enabled=true`.
- **`java`**: reports JVM version and vendor. `management.info.java.enabled=true`.
- **`os`**: reports OS name/version. `management.info.os.enabled=true`.

Custom `InfoContributor`:
```java
@Component
public class AppInfoContributor implements InfoContributor {
    @Override
    public void contribute(Info.Builder builder) {
        builder.withDetail("app", Map.of(
            "name",    "Order Service",
            "env",     "production",
            "tenants", List.of("acme", "globex")
        ));
    }
}
```

Multiple contributors are merged into one JSON response. The `builder.withDetail(key, value)` calls add top-level sections.

## 5. Runnable example

```java
// InfoContributorDemo.java — builds an /actuator/info response from multiple contributors
// How to run: java InfoContributorDemo.java  (JDK 17+, no dependencies)
// Real Spring Boot: @Component class implementing InfoContributor auto-merges into /actuator/info

import java.util.*;

public class InfoContributorDemo {

    // Mirrors Spring's Info.Builder
    static class InfoBuilder {
        final Map<String, Object> details = new LinkedHashMap<>();
        InfoBuilder withDetail(String key, Object value) { details.put(key, value); return this; }
        Map<String, Object> build() { return Collections.unmodifiableMap(details); }
    }

    interface InfoContributor { void contribute(InfoBuilder builder); }

    // === Built-in: BuildInfoContributor (reads build-info.properties) ===
    static InfoContributor buildInfo() {
        return b -> b.withDetail("build", Map.of(
            "artifact", "order-service",
            "version",  "2.4.1",
            "time",     "2025-06-01T10:00:00Z",
            "group",    "com.example"
        ));
    }

    // === Built-in: GitInfoContributor (reads git.properties) ===
    static InfoContributor gitInfo() {
        return b -> b.withDetail("git", Map.of(
            "branch", "main",
            "commit", Map.of(
                "id",        "abc1234",
                "time",      "2025-05-30T08:30:00Z",
                "message",   "feat: add order cancellation"
            ),
            "dirty", false
        ));
    }

    // === Custom: AppInfoContributor ===
    static InfoContributor appInfo(String env, String region) {
        return b -> b.withDetail("app", Map.of(
            "name",     "Order Service",
            "env",      env,
            "region",   region,
            "features", List.of("new-checkout", "ai-search")
        ));
    }

    // === Custom: UpstreamVersionsInfoContributor ===
    static InfoContributor upstreamVersions() {
        return b -> b.withDetail("upstream", Map.of(
            "paymentService",   "v3.2.0",
            "inventoryService", "v1.8.5"
        ));
    }

    static void printJson(Map<String, Object> map, int indent) {
        String pad = " ".repeat(indent);
        map.forEach((k, v) -> {
            if (v instanceof Map<?,?> m) {
                System.out.println(pad + "\"" + k + "\": {");
                printJson((Map<String, Object>) m, indent + 2);
                System.out.println(pad + "},");
            } else {
                System.out.println(pad + "\"" + k + "\": " + (v instanceof String ? "\""+v+"\"" : v) + ",");
            }
        });
    }

    public static void main(String[] args) {
        System.out.println("=== Info Endpoint & InfoContributor Demo ===\n");

        List<InfoContributor> contributors = List.of(
            buildInfo(), gitInfo(), appInfo("production", "eu-west-1"), upstreamVersions()
        );

        // Merge all contributors
        InfoBuilder builder = new InfoBuilder();
        contributors.forEach(c -> c.contribute(builder));
        Map<String, Object> result = builder.build();

        System.out.println("GET /actuator/info");
        System.out.println("{");
        printJson(result, 2);
        System.out.println("}");

        System.out.println("\n--- To enable built-in contributors ---");
        System.out.println("management.info.build.enabled=true   # reads META-INF/build-info.properties");
        System.out.println("management.info.git.enabled=true     # reads git.properties");
        System.out.println("management.info.java.enabled=true    # JVM version");
        System.out.println("management.info.env.enabled=true     # info.* from application.properties");
        System.out.println("\ninfo.app.name=Order Service          # shows in /actuator/info under 'env'");
    }
}
```

**How to run:** `java InfoContributorDemo.java`

## 6. Walkthrough

- **`buildInfo()`**: mirrors `BuildInfoContributor` which reads `META-INF/build-info.properties`. The Maven plugin generates it with `<goal>build-info</goal>`; Gradle uses `bootBuildInfo`.
- **`gitInfo()`**: mirrors `GitInfoContributor` which reads `git.properties`. The `git-commit-id-maven-plugin` generates it. `management.info.git.mode=full` includes the full commit SHA and author.
- **`appInfo`**: a custom `InfoContributor` — add it to expose deployment context (environment, region, feature flags). Useful for support staff who need to identify exactly what's running.
- **`upstreamVersions`**: exposes the versions of downstream services your application talks to. Makes it easy to diagnose version mismatch issues without reading config files.
- All contributors merge into one flat JSON — each `withDetail(key, ...)` call adds a top-level key. No contributor can overwrite another's key (last-write-wins if they share a key).

## 7. Gotchas & takeaways

> `management.info.env.enabled=true` is **required** to expose `info.*` properties — it's disabled by default in Spring Boot 2.6+ for security reasons (properties could contain secrets named `info.*`).

> `build-info.properties` must be generated **at build time** by the Maven/Gradle plugin. Adding the dependency alone is not enough — you must configure the plugin goal.

- Maven: add `<executions><execution><goals><goal>build-info</goal></goals></execution></executions>` inside `spring-boot-maven-plugin`.
- Gradle: call `bootBuildInfo` task (auto-wired in `spring-boot-gradle-plugin`).
- `management.info.git.mode=full` exposes the full SHA instead of just the abbreviated 7-char ID.
- `InfoContributor` beans are ordered via `@Order` — useful if one contributor depends on data from another.
- Automate post-deploy verification: `curl /actuator/info | jq .build.version` and assert it equals the expected version in your CD pipeline.
