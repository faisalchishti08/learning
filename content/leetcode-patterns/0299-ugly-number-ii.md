## 1. What it is

An "ugly number" is a positive integer whose only prime factors are `2`, `3`, and `5`. Given an integer `n`, return the `n`-th ugly number. `1` is conventionally the first ugly number. Example: `n = 10` → `12` (the sequence starts `1,2,3,4,5,6,8,9,10,12`).

## 2. Why & when

This problem generates a sorted sequence from THREE implicit sources: every ugly number, multiplied by `2`, by `3`, or by `5`, produces another ugly number. Merging these three candidate streams, always taking the smallest, produces every ugly number in increasing order — a K-way Merge with `k = 3` GENERATED sequences instead of given ones. Use this shape whenever a sequence is defined by "take known good values, transform them a few different ways, and merge the results."

## 3. Core concept

**Key idea:** maintain the list of ugly numbers found so far, plus three pointers (`p2`, `p3`, `p5`) marking which already-found ugly number each of `2`, `3`, `5` should multiply next. At each step, take the smallest of `ugly[p2]*2`, `ugly[p3]*3`, `ugly[p5]*5` as the next ugly number, and advance whichever pointer(s) produced it.

**Steps:**
1. Start `ugly = [1]`, and `p2 = p3 = p5 = 0`.
2. Repeat `n - 1` times: compute `next2 = ugly[p2]*2`, `next3 = ugly[p3]*3`, `next5 = ugly[p5]*5`; let `next = min(next2, next3, next5)`; append `next` to `ugly`.
3. Advance every pointer whose candidate equalled `next` (advance MORE than one pointer if there is a tie, to avoid inserting duplicates).
4. After the loop, `ugly[n-1]` is the answer.

**Why it is correct:** every ugly number greater than `1` is `2`, `3`, or `5` times some SMALLER ugly number, so scanning candidates generated by multiplying already-found ugly numbers by `2`, `3`, and `5` is guaranteed to eventually produce every ugly number, in order, with no gaps. This is exactly the K-way Merge structure — three sorted "streams" of candidates (`ugly[i]*2`, `ugly[i]*3`, `ugly[i]*5`), merged by always taking the smallest — implemented here with three simple index pointers instead of a heap, since there are only three streams.

Each pointer tracks how far along the "ugly numbers so far" list its own multiplier (2, 3, or 5) has consumed.

## 5. Runnable example

```java
// UglyNumberII.java
public class UglyNumberII {

    // KEY INSIGHT: every ugly number is 2, 3, or 5 times a SMALLER
    // ugly number, so three pointers into the growing ugly-number
    // list act as three merged sorted streams -- no heap needed
    // since there are only 3 streams.

    static int nthUglyNumber(int n) {
        int[] ugly = new int[n];
        ugly[0] = 1;
        int p2 = 0, p3 = 0, p5 = 0;

        for (int i = 1; i < n; i++) {
            int next2 = ugly[p2] * 2;
            int next3 = ugly[p3] * 3;
            int next5 = ugly[p5] * 5;

            int next = Math.min(next2, Math.min(next3, next5));
            ugly[i] = next;

            if (next == next2) p2++;
            if (next == next3) p3++;
            if (next == next5) p5++;
        }
        return ugly[n - 1];
    }

    public static void main(String[] args) {
        System.out.println(nthUglyNumber(10));
        // 12
        System.out.println(nthUglyNumber(1));
        // 1
    }
}
```

**How to run:** `java UglyNumberII.java`

## 6. Walkthrough

Trace `nthUglyNumber(10)`, building `ugly[0..9]`:

| i | next2 | next3 | next5 | next (min) | ugly[i] | pointers advanced |
|---|---|---|---|---|---|---|
| 1 | 2 | 3 | 5 | 2 | 2 | p2-&gt;1 |
| 2 | 4 | 3 | 5 | 3 | 3 | p3-&gt;1 |
| 3 | 4 | 6 | 5 | 4 | 4 | p2-&gt;2 |
| 4 | 6 | 6 | 5 | 5 | 5 | p5-&gt;1 |
| 5 | 6 | 6 | 6 | 6 | 6 | p2-&gt;3, p3-&gt;2, p5-&gt;2 (tie, all three advance) |
| 6 | 8 | 9 | 10 | 8 | 8 | p2-&gt;4 |
| 7 | 9 | 9 | 10 | 9 | 9 | p2-&gt;5, p3-&gt;3 (tie) |
| 8 | 10 | 12 | 10 | 10 | 10 | p2-&gt;6, p5-&gt;3 (tie) |
| 9 | 12 | 12 | 15 | 12 | 12 | p2-&gt;7, p3-&gt;4 (tie) |

`ugly[9] = 12`, the 10th ugly number. Time complexity is O(n), one constant-work step per index. Space is O(n), for the `ugly` array.

## 7. Gotchas & takeaways

> Gotcha: advancing only ONE pointer when `next` ties multiple candidates (using `else if` instead of three separate `if` checks) causes the SAME value to be inserted into `ugly` more than once later, corrupting the sequence — all tied pointers must advance together.

- This is K-way Merge with exactly 3 streams, small enough that three plain index pointers replace a heap entirely — a useful simplification whenever `k` is a small, fixed constant.
- The "generate candidates from what you already found" structure (not a heap merging pre-existing sequences) is what distinguishes this from problems like Merge k Sorted Lists.
- Related problems: Super Ugly Number (the same idea generalized to an arbitrary list of prime factors, needing a heap once the factor count grows).
