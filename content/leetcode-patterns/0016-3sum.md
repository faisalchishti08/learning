## 1. What it is

Given an integer array `nums`, find all unique triplets `[nums[i], nums[j], nums[k]]` (distinct indices) that sum to zero. The result must not contain duplicate triplets. Example: `nums = [-1, 0, 1, 2, -1, -4]` → `[[-1, -1, 2], [-1, 0, 1]]`.

## 2. Why & when

3Sum reduces to Two Sum II: fix one element, then find two others that sum to its negation, in the remaining sorted array — exactly the opposite-ends two-pointers problem, run once per fixed element. Sorting first also gives you a cheap way to skip duplicate triplets.

## 3. Core concept

**Key idea:** for a fixed `nums[i]`, finding two other numbers that sum to `-nums[i]` is Two Sum II on the rest of the (sorted) array.

**Steps:**
1. Sort `nums`.
2. For each index `i` from 0 to `length - 3`:
   - If `nums[i] > 0`, no triplet from here on can sum to zero (all remaining values are `>= nums[i] > 0`) — stop entirely.
   - If `i > 0` and `nums[i] == nums[i - 1]`, skip this `i` to avoid duplicate triplets starting with the same value.
   - Run opposite-ends two pointers on the subarray `[i + 1, length - 1]`, looking for a pair summing to `-nums[i]`.
   - On a match, record the triplet, then move both pointers inward and skip over any duplicate values at the new `left` and `right` before continuing.
3. Return all recorded triplets.

**Why it is correct:** sorting first guarantees that once `nums[i] > 0`, every triplet using it and later elements is also positive, since the array is sorted — so it is safe to stop early. Skipping duplicate values at each of the three positions (`i`, `left`, `right`) avoids emitting the same triplet more than once, without needing a `Set` to deduplicate afterward.

## 5. Runnable example

```java
// ThreeSum.java
import java.util.*;

public class ThreeSum {

    // Level 1 -- Brute force: three nested loops checking every triplet,
    // using a Set<List<Integer>> of sorted triplets to dedupe. O(n^3) time,
    // O(n) space for the dedupe set.
    static List<List<Integer>> bruteForce(int[] nums) {
        Set<List<Integer>> seen = new HashSet<>();
        int n = nums.length;
        for (int i = 0; i < n; i++)
            for (int j = i + 1; j < n; j++)
                for (int k = j + 1; k < n; k++)
                    if (nums[i] + nums[j] + nums[k] == 0) {
                        List<Integer> t = Arrays.asList(nums[i], nums[j], nums[k]);
                        List<Integer> sorted = new ArrayList<>(t);
                        Collections.sort(sorted);
                        seen.add(sorted);
                    }
        return new ArrayList<>(seen);
    }

    // KEY INSIGHT: fixing one element turns 3Sum into Two Sum II on a
    // sorted subarray -- sorting once up front also gives an O(1) way to
    // skip duplicate triplets, with no dedupe set needed.

    // Level 2 -- Optimal: sort + fix + two pointers. O(n^2) time, O(1) extra
    // space beyond the sort and output.
    public static List<List<Integer>> threeSum(int[] nums) {
        Arrays.sort(nums);
        List<List<Integer>> result = new ArrayList<>();
        int n = nums.length;
        for (int i = 0; i < n - 2; i++) {
            if (nums[i] > 0) break;
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            int left = i + 1, right = n - 1;
            int target = -nums[i];
            while (left < right) {
                int sum = nums[left] + nums[right];
                if (sum == target) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    left++;
                    right--;
                    while (left < right && nums[left] == nums[left - 1]) left++;
                    while (left < right && nums[right] == nums[right + 1]) right--;
                } else if (sum < target) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        return result;
    }

    // Level 3 -- Hardened: fewer than 3 elements returns an empty list
    // (the outer loop simply never runs, since n - 2 <= 0).
    static List<List<Integer>> hardened(int[] nums) {
        if (nums == null) throw new IllegalArgumentException("nums must not be null");
        return threeSum(nums);
    }

    public static void main(String[] args) {
        int[] nums = {-1, 0, 1, 2, -1, -4};
        System.out.println("optimal: " + threeSum(nums));
        System.out.println("too small: " + hardened(new int[] {1, 2}));
    }
}
```

How to run: save as `ThreeSum.java`, then run `java ThreeSum.java`.

## 6. Walkthrough

Dry run on sorted `nums = [-4, -1, -1, 0, 1, 2]`, for `i = 1` (`nums[1] = -1`, `target = 1`):

| step | left | right | nums[left] | nums[right] | sum | action |
|---|---|---|---|---|---|---|
| 1 | 2 | 5 | -1 | 2 | 1 | match! add [-1, -1, 2]; left++, right-- |
| 2 | 3 | 4 | 0 | 1 | 1 | match! add [-1, 0, 1]; left++, right-- |
| 3 | 4 | 3 | — | — | — | left >= right, inner loop ends |

For `i = 2` (`nums[2] = -1`), the duplicate-skip check `nums[i] == nums[i - 1]` is true, so it is skipped, avoiding a repeat of the same triplets. Final result: `[[-1, -1, 2], [-1, 0, 1]]`. Time complexity: O(n²) — O(n log n) sort plus O(n) work per fixed `i`. Space complexity: O(1) extra, beyond sort and output.

## 7. Gotchas & takeaways

> Gotcha: forgetting the duplicate-skip checks (on `i`, `left`, and `right`) produces the correct *set* of triplets but with repeats — the problem explicitly requires unique triplets, so all three skip checks are required, not optional cleanup.

- 3Sum is "fix one, Two-Sum-II the rest" — recognizing this reduction is the key insight for the whole *kSum family.
- Related problems: 3Sum Closest, 4Sum, Two Sum II, 3Sum Smaller.
