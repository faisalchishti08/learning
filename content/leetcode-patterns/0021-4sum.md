## 1. What it is

Given an integer array `nums` and an integer `target`, find all unique quadruplets `[nums[a], nums[b], nums[c], nums[d]]` (distinct indices) that sum to `target`. Example: `nums = [1, 0, -1, 0, -2, 2]`, `target = 0` → `[[-2,-1,1,2], [-2,0,0,2], [-1,0,0,1]]`.

## 2. Why & when

4Sum extends the 3Sum reduction one level further: fix **two** elements with nested loops, then run Two Sum II (opposite-ends two pointers) on what remains. Recognizing this generalizes to *k*Sum: fix `k - 2` elements, then two-pointer the last two.

## 3. Core concept

**Key idea:** after sorting, fixing indices `i` and `j` reduces the problem to finding two numbers in the remaining sorted subarray that sum to `target - nums[i] - nums[j]` — exactly Two Sum II.

**Steps:**
1. Sort `nums`.
2. For `i` from 0 to `length - 4`, skipping duplicate values of `nums[i]`:
   - For `j` from `i + 1` to `length - 3`, skipping duplicate values of `nums[j]` (relative to the previous `j` in this `i`'s loop):
     - Run opposite-ends two pointers on `[j + 1, length - 1]` for the target `target - nums[i] - nums[j]`, recording matches and skipping duplicate `left`/`right` values, same as in 3Sum.
3. Return all recorded quadruplets.

**Why it is correct:** the same duplicate-skip logic from 3Sum applies at each nesting level — skip a repeated value for `i`, then for `j` (but only compared to the previous `j` within the same `i`, not across different `i` values), then for `left`/`right` inside the two-pointer scan. This produces every unique quadruplet exactly once, using O(n³) time instead of the O(n⁴) a full four-nested-loop brute force would need.

## 5. Runnable example

```java
// FourSum.java
import java.util.*;

public class FourSum {

    // Level 1 -- Brute force: four nested loops, dedupe with a set of
    // sorted quadruplets. O(n^4) time, O(n) space for the dedupe set.
    static List<List<Integer>> bruteForce(int[] nums, int target) {
        Set<List<Integer>> seen = new HashSet<>();
        int n = nums.length;
        for (int a = 0; a < n; a++)
            for (int b = a + 1; b < n; b++)
                for (int c = b + 1; c < n; c++)
                    for (int d = c + 1; d < n; d++) {
                        long sum = (long) nums[a] + nums[b] + nums[c] + nums[d];
                        if (sum == target) {
                            List<Integer> q = new ArrayList<>(List.of(nums[a], nums[b], nums[c], nums[d]));
                            Collections.sort(q);
                            seen.add(q);
                        }
                    }
        return new ArrayList<>(seen);
    }

    // KEY INSIGHT: fixing two elements with nested loops reduces 4Sum to
    // Two Sum II on the rest -- the same reduction 3Sum uses for one fixed
    // element, applied twice.

    // Level 2 -- Optimal: sort + fix two + two pointers. O(n^3) time, O(1)
    // extra space beyond sort and output.
    public static List<List<Integer>> fourSum(int[] nums, int target) {
        Arrays.sort(nums);
        List<List<Integer>> result = new ArrayList<>();
        int n = nums.length;
        for (int i = 0; i < n - 3; i++) {
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            for (int j = i + 1; j < n - 2; j++) {
                if (j > i + 1 && nums[j] == nums[j - 1]) continue;
                int left = j + 1, right = n - 1;
                long remaining = (long) target - nums[i] - nums[j];
                while (left < right) {
                    long sum = (long) nums[left] + nums[right];
                    if (sum == remaining) {
                        result.add(Arrays.asList(nums[i], nums[j], nums[left], nums[right]));
                        left++;
                        right--;
                        while (left < right && nums[left] == nums[left - 1]) left++;
                        while (left < right && nums[right] == nums[right + 1]) right--;
                    } else if (sum < remaining) {
                        left++;
                    } else {
                        right--;
                    }
                }
            }
        }
        return result;
    }

    // Level 3 -- Hardened: uses `long` for intermediate sums to avoid
    // 32-bit integer overflow when nums contains large values near
    // Integer.MAX_VALUE.
    static List<List<Integer>> hardened(int[] nums, int target) {
        if (nums == null || nums.length < 4) return new ArrayList<>();
        return fourSum(nums, target);
    }

    public static void main(String[] args) {
        int[] nums = {1, 0, -1, 0, -2, 2};
        System.out.println("optimal: " + fourSum(nums, 0));
        System.out.println("too small: " + hardened(new int[] {1, 2, 3}, 6));
    }
}
```

How to run: save as `FourSum.java`, then run `java FourSum.java`.

## 6. Walkthrough

Dry run on sorted `nums = [-2, -1, 0, 0, 1, 2]`, `target = 0`, for `i = 0` (`-2`), `j = 1` (`-1`), so `remaining = 0 - (-2) - (-1) = 3`:

| step | left | right | nums[left] | nums[right] | sum | action |
|---|---|---|---|---|---|---|
| 1 | 2 | 5 | 0 | 2 | 2 | 2 < 3, left++ |
| 2 | 3 | 5 | 0 | 2 | 2 | 2 < 3, left++ |
| 3 | 4 | 5 | 1 | 2 | 3 | match! add [-2,-1,1,2]; left++, right-- |
| 4 | 5 | 4 | — | — | — | left >= right, loop ends |

For `i = 0`, `j = 2` (`nums[2] = 0`), a similar scan finds `[-2, 0, 0, 2]`. Continuing through all `i`, `j` pairs produces the full result: `[[-2,-1,1,2], [-2,0,0,2], [-1,0,0,1]]`. Time complexity: O(n³). Space complexity: O(1) extra.

## 7. Gotchas & takeaways

> Gotcha: computing `nums[i] + nums[j] + nums[left] + nums[right]` with plain `int` arithmetic can overflow if the array holds values near `Integer.MAX_VALUE` or `Integer.MIN_VALUE` — use `long` for the running sums, as the hardened version does.

- The *k*Sum pattern: fix `k - 2` elements with nested loops, then finish with one Two Sum II scan — 2Sum uses a hash map, 3Sum and 4Sum use this reduction, 5Sum would nest one loop deeper still.
- Related problems: 3Sum, 3Sum Closest, Two Sum II, k-Sum (generalized recursive version).
