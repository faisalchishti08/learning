## 1. What it is

Given an array `nums` that MAY contain duplicate values, return all possible subsets (the power set), with NO duplicate subsets in the output. Example: `nums = [1,2,2]` → `[[],[1],[1,2],[1,2,2],[2],[2,2]]`.

## 2. Why & when

This is Subsets with one added twist: duplicate input values would, without special handling, generate the SAME subset multiple times (e.g. picking "the first 2" versus "the second 2" produces an identical-looking subset). Sorting the input first, then skipping over a duplicate value AT THE SAME TREE DEPTH if its identical predecessor was just skipped (not included), eliminates these duplicates.

## 3. Core concept

**Key idea:** sort `nums` so identical values become ADJACENT. In the DFS loop, if the current element equals the PREVIOUS element AND the previous one was NOT included in this branch (i.e., this is not the first time trying a value at this position), skip it — using this value again at the same tree level would reproduce a subset already generated by using the previous occurrence.

**Steps:**
1. Sort `nums` in ascending order (groups duplicate values together).
2. Call the same DFS backtracking helper as Subsets, adding the current partial list at every node.
3. In the loop over `i` from `start` to the end: if `i > start` AND `nums[i] == nums[i-1]`, skip this `i` — this is a duplicate choice at the SAME decision point, which would regenerate a subset already produced.
4. Otherwise, include `nums[i]`, recurse with `i + 1`, then remove `nums[i]` (backtrack) as usual.

**Why it is correct:** sorting groups duplicates together, and skipping a duplicate value when it is NOT the first occurrence being tried AT A GIVEN LOOP (the `i > start` check restricts this to "same level, not the very first option") ensures each distinct multiset of chosen values is generated by exactly one path through the tree — using the FIRST occurrence of a repeated value at each level is always tried, but any LATER occurrence at that same level is redundant, since it would produce an identical subset to the one already generated using the earlier occurrence.

## 5. Runnable example

```java
// SubsetsII.java
import java.util.*;

public class SubsetsII {

    // Level 1 -- Brute force: generate ALL subsets as if elements were
    // distinct (treating duplicate values by their original index),
    // then deduplicate the final result list using a Set<List<Integer>>
    // or by converting each subset to a sorted string key. Correct, but
    // wastes work generating and then discarding many duplicate
    // subsets, instead of preventing them from being generated at all.

    // KEY INSIGHT: sort the input so duplicates are ADJACENT, then skip
    // a repeated value at the SAME tree level (not globally) -- this
    // prevents duplicate subsets from ever being generated, no
    // after-the-fact deduplication needed.

    // Level 2 -- Optimal: sort + skip-adjacent-duplicate-at-same-level.
    static List<List<Integer>> subsetsWithDup(int[] nums) {
        Arrays.sort(nums);
        List<List<Integer>> result = new ArrayList<>();
        dfs(nums, 0, new ArrayList<>(), result);
        return result;
    }

    static void dfs(int[] nums, int start, List<Integer> path, List<List<Integer>> result) {
        result.add(new ArrayList<>(path));
        for (int i = start; i < nums.length; i++) {
            if (i > start && nums[i] == nums[i - 1]) continue;
            path.add(nums[i]);
            dfs(nums, i + 1, path, result);
            path.remove(path.size() - 1);
        }
    }

    // Level 3 -- Hardened: the `i > start` condition (not `i > 0`) is
    // what correctly allows the FIRST occurrence of a duplicate value
    // at each level to still be tried -- only later occurrences at that
    // same level are skipped.

    public static void main(String[] args) {
        System.out.println(subsetsWithDup(new int[]{1,2,2}));
        // [[], [1], [1, 2], [1, 2, 2], [2], [2, 2]]
        System.out.println(subsetsWithDup(new int[]{0}));
        // [[], [0]]
    }
}
```

**How to run:** `java SubsetsII.java`

## 6. Walkthrough

Trace `dfs(nums, 0, [], result)` on sorted `nums = [1,2,2]`:

| Call | path | i tried | Skipped? |
|---|---|---|---|
| dfs(0) | [] | i=0 (value 1) | no, `i==start` |
| dfs(1) via i=0 | [1] | i=1 (value 2) | no, `i==start` |
| dfs(2) via i=1 | [1,2] | i=2 (value 2) | no, `i==start` for this call |
| dfs(3) via i=2 | [1,2,2] | (no more) | — |
| back at dfs(0) | [] | i=1 (value 2) | no, `i==start` |
| dfs(2) via i=1 | [2] | i=2 (value 2, but `nums[2]==nums[1]` AND `i>start`) | YES, skipped |

Skipping the second `2` at the top level (after already trying the first `2` there) prevents generating `[2,2]` a second time via a different path, and prevents a duplicate `[2]`. Time complexity is O(n · 2ⁿ) worst case (same as Subsets, since sorting is O(n log n) and dominated by the generation cost); space is O(n · 2ⁿ) for the output.

## 7. Gotchas & takeaways

> Gotcha: using `i > 0` instead of `i > start` as the skip condition incorrectly skips a duplicate value even when it is the FIRST option being tried at the current recursion level, silently dropping valid subsets that should include that value.

- Sorting first is mandatory — the "adjacent duplicate" skip logic only works when equal values are guaranteed to be next to each other.
- This exact skip-duplicate-at-same-level trick reappears in Combination Sum II and Permutations II — memorize it once, apply it to any "has duplicates in input, no duplicates in output" variant.
- Related problems: Subsets (the no-duplicates base case), Combination Sum II (same skip trick, plus a sum-target constraint), Permutations II (same skip trick, applied to the permutations tree shape instead).
