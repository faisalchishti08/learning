## 1. What it is

`short` is a signed 16-bit two's-complement integer with range **-32,768 to 32,767** inclusive. It is twice the width of `byte` but half the width of `int`.

```java
short year     = 2024;
short port     = (short) 8080;
short minShort = Short.MIN_VALUE;   // -32768
short maxShort = Short.MAX_VALUE;   //  32767

// short s = 32768;   // ✗ compile error: out of range
```

The boxed wrapper is `Short`. Like `byte`, `short` arithmetic always promotes to `int`, and results must be cast back.

## 2. Why & when

`short` is used in narrow circumstances:
- **Memory-critical arrays**: `short[]` uses 2 bytes per element vs 4 for `int[]` — relevant for large datasets like audio samples (16-bit PCM), image channels, or sensor data.
- **Binary protocol compatibility**: some wire formats (e.g., DNS, BMP file headers, class file format) use 16-bit fields.
- **Legacy native interfaces**: JNI code interfacing with C `short` fields.

In everyday code, use `int` instead. The JVM has no native `short` arithmetic instructions — all `short` operations are performed as `int` internally and narrowed on store. `short` saves memory in arrays, not computation time.

## 3. Core concept

```java
// ---- Range ----
System.out.println(Short.MIN_VALUE);    // -32768
System.out.println(Short.MAX_VALUE);    //  32767
System.out.println(Short.SIZE);         // 16 (bits)
System.out.println(Short.BYTES);        // 2

// ---- Arithmetic promotion ----
short a = 1000, b = 2000;
// short c = a + b;     // ✗ compile error: int result
short c = (short)(a + b);  // cast back: 3000 — OK
int   d = a + b;            // widening — preferred

// ---- Overflow ----
short max = Short.MAX_VALUE;   //  32767
max++;                          // wraps to -32768
System.out.println(max);       // -32768

// ---- Unsigned short workaround ----
// short is signed; to treat as unsigned 0–65535:
short s = (short) 0xFFFF;   // -1 as signed
int   u = s & 0xFFFF;        // 65535 as unsigned
System.out.printf("signed=%d  unsigned=%d%n", s, u);

// ---- Byte order for 16-bit fields ----
short value = 0x1234;
byte  high  = (byte)(value >> 8);    // 0x12
byte  low   = (byte)(value & 0xFF);  // 0x34
// Reconstruct big-endian:
short reconstructed = (short)(((high & 0xFF) << 8) | (low & 0xFF));
System.out.println("reconstructed: 0x" + Integer.toHexString(reconstructed & 0xFFFF));

// ---- PCM audio sample (common short use) ----
// 16-bit signed PCM: silence=0, max amplitude=±32767
short silence  = 0;
short fullPos  = Short.MAX_VALUE;
short fullNeg  = Short.MIN_VALUE;
double normalised = fullPos / 32767.0;   // 1.0
System.out.println("normalised: " + normalised);
```

## 5. Runnable example

Scenario: a 16-bit PCM audio order-notification sound system — generate a short sine wave as a `short[]` buffer, normalise it, and encode/decode a binary message header using 16-bit fields.

### Level 1 — Basic

```java
public class ShortBasic {
    public static void main(String[] args) {
        System.out.println("=== short basics ===\n");

        // Range and constants
        System.out.println("Short.MIN_VALUE: " + Short.MIN_VALUE);
        System.out.println("Short.MAX_VALUE: " + Short.MAX_VALUE);
        System.out.println("Short.SIZE:      " + Short.SIZE + " bits");
        System.out.println("Short.BYTES:     " + Short.BYTES + " bytes");

        // Arithmetic promotion
        short x = 10_000, y = 5_000;
        // short z = x + y;     // ✗ compile error
        short z = (short)(x + y);  // 15000 — fits
        System.out.printf("%n10000 + 5000 = %d (as short)%n", z);

        // Overflow
        short maxVal = Short.MAX_VALUE;
        System.out.printf("32767 + 1 = %d (short overflow)%n", (short)(maxVal + 1));

        // Unsigned interpretation
        short neg = -1;
        int unsigned = neg & 0xFFFF;
        System.out.printf("%n-1 as short  = 0x%04X%n", neg & 0xFFFF);
        System.out.printf("-1 & 0xFFFF  = %d (unsigned)%n", unsigned);

        // PCM audio: 16-bit silence, max amplitude
        short silence = 0;
        short maxAmp  = Short.MAX_VALUE;   //  32767
        short minAmp  = Short.MIN_VALUE;   // -32768
        System.out.printf("%nPCM silence=%d  max=%d  min=%d%n", silence, maxAmp, minAmp);

        // Normalise to -1.0..1.0 range (standard audio)
        System.out.printf("Normalised max: %.4f%n", maxAmp / 32767.0);
        System.out.printf("Normalised min: %.4f%n", minAmp / 32768.0);
    }
}
```

**How to run:** `java ShortBasic.java`

`(short)(x + y)` is necessary because `x + y` produces an `int`. When the result fits in -32768..32767, the cast is safe. If it doesn't, it silently wraps — the same behaviour as `byte` overflow.

### Level 2 — Intermediate

Same sound system: generate a 16-bit PCM sine wave, store samples in a `short[]`, and compute basic statistics (peak, RMS, zero crossings).

```java
public class ShortIntermediate {
    public static void main(String[] args) {
        System.out.println("=== short: 16-bit PCM sine wave ===\n");

        int sampleRate = 44_100;     // 44.1 kHz
        int durationMs = 100;        // 100 ms
        double freqHz  = 440.0;      // A4 note
        double amp     = 0.8;        // 80% amplitude

        int numSamples = sampleRate * durationMs / 1000;
        short[] pcm = new short[numSamples];

        // Generate sine wave
        for (int i = 0; i < numSamples; i++) {
            double t    = (double) i / sampleRate;
            double sine = Math.sin(2.0 * Math.PI * freqHz * t);
            pcm[i] = (short)(sine * amp * Short.MAX_VALUE);
        }

        System.out.printf("Sample rate: %,d Hz%n", sampleRate);
        System.out.printf("Duration:    %d ms%n", durationMs);
        System.out.printf("Samples:     %,d%n", numSamples);
        System.out.printf("Array size:  %,d bytes (short[] = 2 bytes/sample)%n",
            numSamples * Short.BYTES);

        // Statistics
        short peak = 0;
        double sumSquares = 0;
        int zeroCrossings = 0;
        for (int i = 0; i < pcm.length; i++) {
            short s = pcm[i];
            if (Math.abs(s) > Math.abs(peak)) peak = s;
            sumSquares += (double) s * s;
            if (i > 0 && ((pcm[i-1] >= 0 && pcm[i] < 0) || (pcm[i-1] < 0 && pcm[i] >= 0)))
                zeroCrossings++;
        }
        double rms = Math.sqrt(sumSquares / pcm.length);

        System.out.printf("%nStatistics:%n");
        System.out.printf("  Peak sample:   %d (%.1f%% of max)%n",
            peak, 100.0 * peak / Short.MAX_VALUE);
        System.out.printf("  RMS:           %.1f%n", rms);
        System.out.printf("  Zero crossings:%d (expected ~%d for %.0f Hz)%n",
            zeroCrossings, (int)(2 * freqHz * durationMs / 1000.0), freqHz);

        // Print first 8 samples
        System.out.println("\nFirst 8 samples:");
        for (int i = 0; i < 8; i++)
            System.out.printf("  [%d] %6d  (0x%04X)%n", i, pcm[i], pcm[i] & 0xFFFF);
    }
}
```

**How to run:** `java ShortIntermediate.java`

`(short)(sine * amp * Short.MAX_VALUE)` — `sine` is in [-1, 1], `amp` scales it to 80%, multiplied by 32767 gives a range of ±26213. The cast to `short` stores it in 2 bytes. A `double[]` would use 8 bytes per sample — 4× the memory for the same data.

### Level 3 — Advanced

Same system: encode/decode a binary message header with multiple 16-bit fields using `ByteBuffer`, compare `short[]` vs `int[]` memory footprint, and demonstrate `DataOutputStream`/`DataInputStream` for typed binary I/O.

```java
import java.io.*;
import java.nio.*;
import java.util.*;

public class ShortAdvanced {
    // Binary header: 10 bytes
    // [0–1]  magic  (0xF00D as short)
    // [2–3]  version
    // [4–5]  itemCount
    // [6–7]  statusCode
    // [8–9]  checksum (XOR of fields)

    record Header(short magic, short version, short itemCount, short statusCode) {
        short checksum() {
            return (short)(magic ^ version ^ itemCount ^ statusCode);
        }
    }

    static byte[] encode(Header h) {
        ByteBuffer bb = ByteBuffer.allocate(10).order(ByteOrder.BIG_ENDIAN);
        bb.putShort(h.magic());
        bb.putShort(h.version());
        bb.putShort(h.itemCount());
        bb.putShort(h.statusCode());
        bb.putShort(h.checksum());
        return bb.array();
    }

    static Header decode(byte[] buf) throws IOException {
        ByteBuffer bb = ByteBuffer.wrap(buf).order(ByteOrder.BIG_ENDIAN);
        short magic      = bb.getShort();
        short version    = bb.getShort();
        short itemCount  = bb.getShort();
        short statusCode = bb.getShort();
        short checksum   = bb.getShort();
        var h = new Header(magic, version, itemCount, statusCode);
        if (checksum != h.checksum())
            throw new IOException("Checksum mismatch: got " + checksum + " expected " + h.checksum());
        return h;
    }

    public static void main(String[] args) throws Exception {
        System.out.println("=== Advanced short: binary header encode/decode ===\n");

        // 1. Encode/decode
        var original = new Header((short) 0xF00D, (short) 3, (short) 127, (short) 0x0001);
        byte[] encoded = encode(original);

        System.out.println("[ Header encode ]");
        System.out.printf("  magic=0x%04X  version=%d  items=%d  status=0x%04X%n",
            original.magic() & 0xFFFF, original.version(), original.itemCount(),
            original.statusCode() & 0xFFFF);
        System.out.printf("  Encoded (%d bytes): ", encoded.length);
        for (byte b : encoded) System.out.printf("%02X ", b & 0xFF);
        System.out.println();

        Header decoded = decode(encoded);
        System.out.printf("  Decoded: magic=0x%04X  version=%d  items=%d  status=0x%04X%n",
            decoded.magic() & 0xFFFF, decoded.version(), decoded.itemCount(),
            decoded.statusCode() & 0xFFFF);
        System.out.println("  Checksum OK: " + (decoded.checksum() == original.checksum()));

        // 2. Memory comparison: short[] vs int[]
        System.out.println("\n[ Memory: short[] vs int[] ]");
        int n = 1_000_000;
        short[] shortArr = new short[n];
        int[]   intArr   = new int[n];
        // Approximate JVM object sizes (overhead + data)
        long shortBytes = 16 + (long) n * Short.BYTES;
        long intBytes   = 16 + (long) n * Integer.BYTES;
        System.out.printf("  short[%,d]: ~%,d bytes (%,d KB)%n", n, shortBytes, shortBytes/1024);
        System.out.printf("  int[%,d]:   ~%,d bytes (%,d KB)%n", n, intBytes,   intBytes/1024);
        System.out.printf("  Saving:     %,d bytes (%,d KB)%n",
            intBytes - shortBytes, (intBytes-shortBytes)/1024);

        // 3. DataOutputStream round-trip
        System.out.println("\n[ DataOutputStream/DataInputStream ]");
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        DataOutputStream dos = new DataOutputStream(baos);
        short[] samples = {-32768, -1000, 0, 1000, 32767};
        dos.writeShort(samples.length);
        for (short s : samples) dos.writeShort(s);
        dos.close();

        byte[] data = baos.toByteArray();
        System.out.printf("  Written %d shorts + length = %d bytes%n",
            samples.length, data.length);

        DataInputStream dis = new DataInputStream(new ByteArrayInputStream(data));
        int count = dis.readShort() & 0xFFFF;
        System.out.printf("  Read back %d samples: ", count);
        for (int i = 0; i < count; i++) System.out.printf("%d ", dis.readShort());
        System.out.println();
        dis.close();
    }
}
```

**How to run:** `java ShortAdvanced.java`

`dos.writeShort(s)` writes 2 bytes in big-endian order — the same byte order used by Java's class file format, DNS, and most network protocols. `dis.readShort()` reads them back as a signed `short`; `& 0xFFFF` converts to unsigned if needed.

## 6. Walkthrough

Execution trace in `ShortAdvanced.main`:

**`encode(original)`.** `ByteBuffer.allocate(10).order(ByteOrder.BIG_ENDIAN)` creates a 10-byte buffer in network byte order (most significant byte first). `bb.putShort((short) 0xF00D)` writes bytes `0xF0 0x0D`. `bb.putShort((short) 127)` writes `0x00 0x7F`. The checksum `(short)(0xF00D ^ 3 ^ 127 ^ 1)` is XOR of all four fields — a simple integrity check.

**`decode(buf)`.** `bb.getShort()` reads each 2-byte field in order. After reading all four header fields, `h.checksum()` recomputes the XOR. If it doesn't match the stored checksum, the data was corrupted — `IOException` is thrown. The `& 0xFFFF` in the print statement converts signed `short` to unsigned `int` for hex display: `(short)0xF00D` is -4083 as signed; `0xF00D & 0xFFFF` is 61453.

**Memory comparison.** `short[]` uses `n * 2` bytes of element data; `int[]` uses `n * 4`. For 1 million elements, `short[]` saves 2 MB. At this scale the difference matters for: JVM GC pressure (fewer live bytes), CPU cache efficiency (more elements per cache line), and heap size.

**`DataOutputStream.writeShort`** writes high byte then low byte (big-endian), regardless of platform. `DataInputStream.readShort` reads and reconstructs the 16-bit value. This is why Java I/O streams are platform-independent for binary data — byte order is fixed by the API.

## 7. Gotchas & takeaways

> **`short` arithmetic always promotes to `int`.** This is true even for `short s = 1; s += 1;` — the compiler inserts a hidden cast: `s = (short)(s + 1)`. Using `short` in a tight loop is not faster than `int`; the JVM has no native 16-bit arithmetic registers. Use `short` only where the storage saving in `short[]` matters.

> **`0xFFFF` values are negative as `short`.** `(short) 0x8000` is -32768; `(short) 0xFFFF` is -1. Always use `& 0xFFFF` when you need the unsigned 0–65535 interpretation of a 16-bit field from a binary protocol.

- Range: -32,768 to 32,767. `Short.MIN_VALUE` / `MAX_VALUE`. `Short.BYTES = 2`.
- Arithmetic promotes `short` to `int`; cast result back explicitly: `(short)(a + b)`.
- Overflow wraps silently: `(short)(32767 + 1) == -32768`.
- Unsigned 0–65535: `shortVar & 0xFFFF` (clears sign-extension bits).
- Use `short[]` for 16-bit PCM audio, large in-memory tables where memory matters.
- `ByteBuffer.putShort`/`getShort` and `DataOutputStream.writeShort`/`DataInputStream.readShort` handle binary I/O with correct byte order.
- In everyday code, use `int`. Only reach for `short` when `short[]` memory saving is measurable.
