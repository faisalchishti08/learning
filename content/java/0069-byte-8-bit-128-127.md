## 1. What it is

`byte` is Java's smallest integer primitive — a signed 8-bit two's-complement integer with the range **-128 to 127** inclusive. It stores one byte of data: 256 possible values.

```java
byte small  =  100;
byte min    = -128;
byte max    =  127;
// byte overflow = 128;   // ✗ compile error: 128 is out of range

byte fromHex = (byte) 0xFF;   // must cast — 0xFF = 255 as int; cast wraps to -1
```

The boxed wrapper is `Byte`. `byte[]` is the standard type for binary I/O, network buffers, and raw data.

## 2. Why & when

`byte` exists for two purposes:
1. **Memory efficiency in arrays**: a `byte[]` storing a million elements uses 1 MB, compared to 4 MB for `int[]`. This matters for file I/O buffers, image pixel data, and network packet payloads.
2. **Binary protocol handling**: network/file formats work in octets (8-bit units). `byte` maps directly to the native unit, though you must be careful about Java's signed representation vs protocol unsigned values.

In everyday arithmetic, prefer `int` — the JVM promotes `byte` operands to `int` for all arithmetic anyway, so `byte` arithmetic doesn't save computation time, only storage.

## 3. Core concept

```java
// ---- Range and two's complement ----
// Bit pattern:   00000000 → 0
//                01111111 → 127   (max positive)
//                10000000 → -128  (min: the sign bit flips)
//                11111111 → -1

byte max  = Byte.MAX_VALUE;   //  127
byte min  = Byte.MIN_VALUE;   // -128
System.out.println(Byte.SIZE);       // 8  (bits)
System.out.println(Byte.BYTES);      // 1  (bytes)

// ---- Arithmetic promotion ----
byte a = 10, b = 20;
// byte c = a + b;    // ✗ compile error: result is int, not byte
byte c = (byte)(a + b);   // explicit cast back to byte
int  d = a + b;           // ✓ widening — int holds result safely

// ---- Overflow (wrap-around) ----
byte wrap = 127;
wrap++;           // wrap becomes -128 (two's complement roll-over)
System.out.println(wrap);  // -128

// ---- Literal assignment ----
byte lit = 42;             // constant 42 fits in byte — OK without cast
byte lit2 = (byte) 200;    // 200 > 127; cast required; stores -56

// ---- Unsigned interpretation ----
// Java byte is SIGNED. Protocols often use UNSIGNED bytes (0–255).
// Workaround: mask to int with & 0xFF
byte signed = (byte) 0xAB;           // -85 (signed)
int  unsigned = signed & 0xFF;        // 171 (unsigned view)
System.out.printf("signed=%d  unsigned=%d%n", signed, unsigned);

// ---- I/O: InputStream.read() returns int, not byte ----
// InputStream.read() returns -1 on EOF, 0–255 for data.
// If it returned byte, -1 (EOF) would clash with valid byte value -1.
int byteRead = System.in.read();  // returns int
if (byteRead != -1) {
    byte b2 = (byte) byteRead;   // safe: we know 0–255
}
```

## 5. Runnable example

Scenario: a binary network packet encoder/decoder for an order system — packs order metadata into a byte array, reads it back, and handles signed/unsigned conversion.

### Level 1 — Basic

```java
public class ByteBasic {
    public static void main(String[] args) {
        System.out.println("=== byte basics ===\n");

        // Range
        System.out.println("Byte.MIN_VALUE: " + Byte.MIN_VALUE);   // -128
        System.out.println("Byte.MAX_VALUE: " + Byte.MAX_VALUE);   //  127
        System.out.println("Byte.SIZE:      " + Byte.SIZE + " bits");
        System.out.println("Byte.BYTES:     " + Byte.BYTES + " byte");

        // Literal assignment
        byte status  = 1;     // 1 = paid
        byte version = 3;
        byte flags   = 0b00000101;  // binary literal: bits 0 and 2 set

        System.out.printf("%nstatus=%-4d  version=%-4d  flags=%d (binary: %s)%n",
            status, version, flags, Integer.toBinaryString(flags & 0xFF));

        // Overflow (wrap-around)
        byte b = 127;
        b++;           // 127 + 1 overflows → -128
        System.out.println("127++ = " + b);

        b = -128;
        b--;           // -128 - 1 overflows → 127
        System.out.println("-128-- = " + b);

        // Arithmetic promotion: byte + byte = int
        byte x = 10, y = 20;
        // byte z = x + y;     // ✗ compile error
        byte z = (byte)(x + y);  // cast back
        int  w = x + y;          // widening — safe
        System.out.printf("%nbyte + byte: z=%d (byte) w=%d (int)%n", z, w);

        // Unsigned interpretation
        byte signedFF = (byte) 0xFF;  // -1 as signed
        int unsignedFF = signedFF & 0xFF;   // 255 as unsigned
        System.out.printf("%n0xFF as byte: %d (signed)  %d (unsigned via & 0xFF)%n",
            signedFF, unsignedFF);
    }
}
```

**How to run:** `java ByteBasic.java`

`b++` on `byte` 127 wraps to -128 because the JVM performs the increment as `int` arithmetic, then narrows back to `byte`. 128 in binary is `10000000`, which in two's-complement 8-bit is -128.

### Level 2 — Intermediate

Same packet encoder: build a minimal binary packet for order status — pack fields into a `byte[]`, read them back, and handle unsigned byte values from a simulated network stream.

```java
public class ByteIntermediate {
    public static void main(String[] args) {
        System.out.println("=== byte: binary packet encode/decode ===\n");

        // Packet format (8 bytes):
        // [0]     version (1 byte, unsigned: 0–255)
        // [1]     status  (1 byte: 0=pending, 1=paid, 2=shipped)
        // [2–3]   item count (2 bytes, big-endian, unsigned 0–65535)
        // [4–7]   amount_cents (4 bytes, big-endian, signed int)

        int version    = 2;
        int status     = 1;    // paid
        int itemCount  = 300;  // 300 items (won't fit in a single byte)
        int amountCents = 29999;  // £299.99

        byte[] packet = new byte[8];
        packet[0] = (byte) version;    // 2 fits, no sign issue
        packet[1] = (byte) status;     // 1 fits

        // item count: 300 = 0x012C → high byte 0x01, low byte 0x2C
        packet[2] = (byte)(itemCount >> 8);   // high byte
        packet[3] = (byte)(itemCount & 0xFF); // low byte

        // amount: 4 bytes big-endian
        packet[4] = (byte)(amountCents >> 24);
        packet[5] = (byte)(amountCents >> 16);
        packet[6] = (byte)(amountCents >> 8);
        packet[7] = (byte)(amountCents);

        System.out.println("Encoded packet bytes:");
        for (int i = 0; i < packet.length; i++) {
            System.out.printf("  [%d] %4d (0x%02X)%n", i, packet[i], packet[i] & 0xFF);
        }

        // Decode
        int dVersion    = packet[0] & 0xFF;
        int dStatus     = packet[1] & 0xFF;
        int dItemCount  = ((packet[2] & 0xFF) << 8) | (packet[3] & 0xFF);
        int dAmount     = (packet[4] << 24) | ((packet[5] & 0xFF) << 16)
                        | ((packet[6] & 0xFF) << 8) | (packet[7] & 0xFF);

        System.out.printf("%nDecoded:%n");
        System.out.printf("  version:     %d%n", dVersion);
        System.out.printf("  status:      %d (%s)%n", dStatus,
            dStatus == 0 ? "pending" : dStatus == 1 ? "paid" : "shipped");
        System.out.printf("  items:       %d%n", dItemCount);
        System.out.printf("  amount:      £%.2f%n", dAmount / 100.0);

        // Verify round-trip
        System.out.printf("%nRound-trip OK: %b%n",
            dVersion == version && dStatus == status
            && dItemCount == itemCount && dAmount == amountCents);
    }
}
```

**How to run:** `java ByteIntermediate.java`

`(packet[2] & 0xFF) << 8` — the `& 0xFF` is essential: `packet[2]` is a `byte` (sign-extended to `int` when used in arithmetic). For values ≥ 128, sign extension would set the upper bits, corrupting the shift. `& 0xFF` clears the upper 24 bits before shifting.

### Level 3 — Advanced

Same order system: use `java.io.DataInputStream`/`DataOutputStream` for structured binary I/O, benchmark `byte[]` buffer read speed, and show `ByteBuffer` (NIO) as the modern alternative.

```java
import java.io.*;
import java.nio.*;
import java.nio.charset.StandardCharsets;
import java.util.*;

public class ByteAdvanced {
    record OrderPacket(int version, int status, int items, int amountCents, String id) {}

    static final int PACKET_SIZE = 20;  // bytes

    static byte[] encode(OrderPacket p) {
        byte[] buf = new byte[PACKET_SIZE];
        ByteBuffer bb = ByteBuffer.wrap(buf).order(ByteOrder.BIG_ENDIAN);
        bb.put((byte) p.version());
        bb.put((byte) p.status());
        bb.putShort((short) p.items());       // 2 bytes
        bb.putInt(p.amountCents());           // 4 bytes
        byte[] idBytes = p.id().getBytes(StandardCharsets.UTF_8);
        bb.put((byte) idBytes.length);        // 1 byte: length prefix
        bb.put(Arrays.copyOf(idBytes, Math.min(idBytes.length, 12))); // up to 12 bytes
        return buf;
    }

    static OrderPacket decode(byte[] buf) {
        ByteBuffer bb = ByteBuffer.wrap(buf).order(ByteOrder.BIG_ENDIAN);
        int version      = bb.get() & 0xFF;
        int status       = bb.get() & 0xFF;
        int items        = bb.getShort() & 0xFFFF;
        int amountCents  = bb.getInt();
        int idLen        = bb.get() & 0xFF;
        byte[] idBytes   = new byte[Math.min(idLen, 12)];
        bb.get(idBytes);
        return new OrderPacket(version, status, items, amountCents,
                               new String(idBytes, StandardCharsets.UTF_8));
    }

    public static void main(String[] args) throws Exception {
        System.out.println("=== Advanced byte: ByteBuffer encode/decode + benchmark ===\n");

        // 1. Encode/decode round-trip
        OrderPacket original = new OrderPacket(2, 1, 300, 29999, "ORD-001");
        byte[] encoded = encode(original);
        OrderPacket decoded = decode(encoded);

        System.out.println("[ Encode/decode round-trip ]");
        System.out.printf("  Original: %s%n", original);
        System.out.printf("  Encoded (%d bytes): ", encoded.length);
        for (byte b : encoded) System.out.printf("%02X ", b & 0xFF);
        System.out.printf("%n  Decoded:  %s%n", decoded);
        System.out.printf("  Match: %b%n", original.equals(decoded));

        // 2. Benchmark: byte[] vs ByteBuffer
        System.out.println("\n[ byte[] I/O benchmark ]");
        int iterations = 10_000;
        var orders = new byte[iterations][PACKET_SIZE];

        long t1 = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            OrderPacket p = new OrderPacket(1, i % 3, i % 500, i * 100, "ORD");
            orders[i] = encode(p);
        }
        long t2 = System.nanoTime();

        int decoded2 = 0;
        for (byte[] buf : orders) { decode(buf); decoded2++; }
        long t3 = System.nanoTime();

        System.out.printf("  Encoded %,d packets: %,d µs%n", iterations, (t2-t1)/1000);
        System.out.printf("  Decoded %,d packets: %,d µs%n", decoded2,   (t3-t2)/1000);

        // 3. Signed vs unsigned summary
        System.out.println("\n[ Signed vs unsigned byte summary ]");
        int[] values = {0, 1, 127, 128, 200, 255};
        System.out.printf("  %-6s  %-8s  %-8s  %-8s%n", "value", "(byte)", "& 0xFF", "hex");
        for (int v : values) {
            byte signed = (byte) v;
            System.out.printf("  %-6d  %-8d  %-8d  0x%02X%n",
                v, signed, signed & 0xFF, v);
        }
    }
}
```

**How to run:** `java ByteAdvanced.java`

`ByteBuffer.wrap(buf)` creates a view over the `byte[]` without copying. `bb.putInt(amountCents)` writes 4 bytes in big-endian order directly. `bb.getShort() & 0xFFFF` reads a signed `short` and widens it to an unsigned `int` — the same `& mask` pattern for clearing sign-extension bits.

## 6. Walkthrough

Execution trace in `ByteAdvanced.main`:

**`encode(OrderPacket)`.** `ByteBuffer.wrap(buf)` wraps the 20-byte array. Each `put`/`putShort`/`putInt` call advances the buffer's internal position. `(byte) p.version()` narrows `int 2` to byte `0x02`. `bb.putShort((short) 300)` writes `0x01 0x2C`. `bb.putInt(29999)` writes `0x00 0x00 0x75 0x2F`. The ID `"ORD-001"` is encoded as UTF-8 bytes with a 1-byte length prefix.

**`decode(byte[])`.** `bb.get() & 0xFF` reads one byte and clears the sign-extension bits. `bb.getShort() & 0xFFFF` reads two bytes as a signed `short` (possibly negative if bit 15 set) and widens to a non-negative `int` by masking. `bb.getInt()` reads four bytes as a signed `int` directly — no masking needed since `int` is the natural size.

**Hex dump.** `packet[i] & 0xFF` is required for `%02X` formatting: `%X` formats an `int`, and `byte` widens to `int` with sign extension — `(byte) 0xFF` = -1 = `0xFFFFFFFF` as `int`. The mask clears the upper bits, giving `0x000000FF` which prints as `FF`.

## 7. Gotchas & takeaways

> **`byte` is signed: values ≥ 128 are negative.** `(byte) 200` is -56. When reading binary data, always use `byteVar & 0xFF` to get the unsigned 0–255 interpretation. Forgetting this causes subtle bugs when comparing or printing byte values from file/network I/O.

> **`InputStream.read()` returns `int`, not `byte`.** This is intentional: the method returns -1 for end-of-stream, which must not be confused with the valid byte value -1 (0xFF). Always check `!= -1` before casting to `byte`.

- Range: -128 to 127 (signed 8-bit two's complement). `Byte.MIN_VALUE`/`MAX_VALUE`.
- Arithmetic promotes `byte` to `int` — cast result back to `byte` explicitly: `(byte)(a + b)`.
- Overflow wraps silently: `(byte)(127 + 1) == -128`.
- Unsigned 0–255: `byteVar & 0xFF` clears sign-extension bits after widening.
- `byte[]` is the standard type for binary I/O buffers, image data, network payloads.
- Use `ByteBuffer` (NIO) for structured binary encoding/decoding — handles byte order and typed puts/gets.
- Don't use `byte` for ordinary arithmetic — use `int`. Use `byte` for arrays and binary protocols.
